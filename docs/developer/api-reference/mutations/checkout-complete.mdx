Completes the checkout. As a result a new order is created and a payment charge is made. This action requires a successful payment before it can be performed. In case additional confirmation step as 3D secure is required confirmationNeeded flag will be set to True and no order created until payment is confirmed with second call of this mutation.

```graphql
checkoutComplete(
  checkoutId: ID
  id: ID
  metadata: [MetadataInput!]
  paymentData: JSONString
  redirectUrl: String
  storeSource: Boolean = false
  token: UUID
): CheckoutComplete
```

### Arguments

#### [`checkoutId`](#) ([`ID`](../../../developer/api-reference/scalars/id))

The ID of the checkout.

DEPRECATED: this field will be removed in Saleor 4.0. Use `id` instead.

#### [`id`](#) ([`ID`](../../../developer/api-reference/scalars/id))

The checkout's ID.

Added in Saleor 3.4.

#### [`metadata`](#) ([`[MetadataInput!]`](../../../developer/api-reference/inputs/metadata-input))

Fields required to update the checkout metadata.

Added in Saleor 3.8.

#### [`paymentData`](#) ([`JSONString`](../../../developer/api-reference/scalars/jsonstring))

Client-side generated data required to finalize the payment.

#### [`redirectUrl`](#) ([`String`](../../../developer/api-reference/scalars/string))

URL of a view where users should be redirected to see the order details. URL in RFC 1808 format.

#### [`storeSource`](#) ([`Boolean`](../../../developer/api-reference/scalars/boolean))

Determines whether to store the payment source for future usage.

DEPRECATED: this field will be removed in Saleor 4.0. Use checkoutPaymentCreate for this action.

#### [`token`](#) ([`UUID`](../../../developer/api-reference/scalars/uuid))

Checkout token.

DEPRECATED: this field will be removed in Saleor 4.0. Use `id` instead.
